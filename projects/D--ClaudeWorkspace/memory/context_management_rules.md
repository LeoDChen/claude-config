## 大任务分对话执行

抓取/分析等大任务必须在不同对话中执行。不要把诊断、探索、抓取、分析全塞进一个对话。

- 完成脚本编写后，告知用户脚本路径和用法，提醒在**新对话**中执行
- 不要在同一对话中既抓取又分析
- 每次新对话开始时，先读 STATE.md 和 knowledge/ 了解状态

**Why:** 单对话持续做多阶段任务会导致上下文膨胀，模型输出和执行质量下降。

## 主动评估容量，提前拆分

长任务（如OKKI客户梯队分析）在执行前必须评估当前对话上下文状态。不能等上下文爆了再拆分。

判断标准：已做 3 个以上大步骤 + 工具调用 > 50 次 → 考虑切新对话。切之前把进度写入 STATE.md + 生成 NEXT_BATCH.md 启动指令。拆分后每个新对话任务量 ≤ 本对话已完成量的 60%。

**Why:** 2026-05-09 OKKI 分析在同一对话塞入同步→分类→15人分析→Excel→STATE→审计纠偏后，上下文已拥挤，再塞深度分析导致输出质量下降。
