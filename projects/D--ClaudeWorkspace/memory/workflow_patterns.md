## 常见任务工作流程

### 1. 文件分析任务
```
用户提供文件 → AI读取内容 → 提取关键信息 → 生成分析报告 → 提供建议
```
**适用场景：** 市场报告分析、销售数据解读、产品目录整理

### 2. 运营策略制定
```
明确目标 → 市场调研 → 竞品分析 → 策略设计 → 实施步骤 → 风险控制
```
**适用场景：** 跨境电商运营、市场进入策略、营销方案制定

### 3. 文档创建/优化
```
了解需求 → 收集素材 → 结构设计 → 内容撰写 → 格式优化 → 反馈修改
```
**适用场景：** 产品详情页优化、营销文案、操作手册、报告撰写

### 4. 数据分析
```
获取数据文件 → 数据清洗 → 分析指标确定 → 可视化呈现 → 洞察提炼
```
**适用场景：** 销售数据分析、流量分析、客户行为分析

### 5. 问题解决
```
问题描述 → 原因分析 → 方案设计 → 实施步骤 → 验证方法
```
**适用场景：** 技术问题排查、运营障碍解决、流程优化

## 最佳实践

### 沟通效率
- **一次一问**：将复杂任务分解为多个明确的小问题
- **提供上下文**：分享相关文件、截图、数据
- **明确期望**：说明希望得到的输出格式（表格、列表、代码等）

### 文件管理
- **项目文件夹**：按项目创建专用文件夹，存放相关文件
- **命名规范**：使用清晰的文件名，如 `市场分析_苏丹_202604.docx`
- **版本控制**：重要文件添加日期版本号

### 协作习惯
- **及时反馈**：对AI的建议提供明确反馈（肯定/修正）
- **积累记忆**：重要偏好和决策保存到记忆系统
- **渐进优化**：从简单任务开始，逐步增加复杂度
