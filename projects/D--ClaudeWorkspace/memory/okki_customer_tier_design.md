# 客户等级判定（待实现）

## 核心结论

**能做。有了准确状态卡之后，等级判定比之前准得多。** 不是AI猜测，是从状态卡字段直接算。

## 可用的判定维度

| 维度 | 来源字段 | 说明 |
|------|---------|------|
| 订单量/金额 | 关键数字 | 客户采购规模 |
| 买家类型 | 画像 | 批发商 vs 小零售商 vs 品牌商 |
| 复购次数 | 阶段+历史 | 忠诚度/稳定性 |
| 谈判深度 | 阶段 | 意向真实度 |
| 互动频率/质量 | 聊天摘要 | 关系健康度 |
| 国家 | 画像 | 市场匹配度（核心市场 vs 边缘） |

## 展示方案（推荐）

**方案 A + D 组合**：
1. 看板矩阵保持阶段×紧急度，不做第三维
2. chip上加等级标记：A级=粗体+★前缀，B级=正常，C级=灰色小字
3. 顶部加等级筛选按钮：[全部] [★A级] [☆B级] [C级]
4. 等级从状态卡字段自动算，不靠AI猜

## 为什么不是 NOW/SOON/COLD 的替代

- 热度/紧急度 = "现在该做什么"（行动信号，会变）
- 等级 = "这个客户值多少钱"（商业价值，相对稳定）
- 两个是不同的维度，不互相替代
